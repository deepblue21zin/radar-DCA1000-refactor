1. 메인 창과 패널을 만든다

Step

setupUi는 QMainWindow 안에 central widget, plot 영역, 버튼, 라벨을 배치한다.

왜 필요한가실시간 앱은 먼저 표시 영역과 제어 버튼 골격이 있어야 한다.
다음으로 넘기는 것위젯 트리

2. objectName을 고정한다

Step

각 위젯의 이름이 지정되어 상위 코드가 해당 객체를 찾아 쓸 수 있게 된다.

왜 필요한가UI 파일과 로직 파일이 분리되어 있을 때 objectName이 사실상 계약 역할을 한다.
다음으로 넘기는 것live_motion_viewer가 참조할 UI 식별자

3. 표시 텍스트를 채운다

Step

retranslateUi가 버튼과 레이블 텍스트를 최종 세팅한다.

왜 필요한가다국어 대응이나 텍스트 변경도 구조와 분리해 두기 위해서다.
다음으로 넘기는 것사용자에게 보이는 기본 UI 텍스트

놓치기 쉬운 부분

코드를 읽을 때 자주 헷갈리거나, 설명 없이 넘어가면 오해하기 쉬운 부분을 따로 정리했다.

  • 로직이 적다고 중요하지 않은 파일은 아니다. objectName이 바뀌면 상위 컨트롤러가 바로 깨질 수 있다.
  • Qt Designer generated file 성격이라 직접 수정한 내용은 재생성 때 덮일 수 있다.
  • 이 파일은 데이터 처리 코드가 아니라 UI 계약 파일로 읽는 편이 좋다.
  • 화면 문제를 디버깅할 때는 이 파일과 live_motion_viewer.py를 같이 봐야 한다.
