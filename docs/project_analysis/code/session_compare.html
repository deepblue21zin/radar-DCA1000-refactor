놓치기 쉬운 부분

코드를 읽을 때 자주 헷갈리거나, 설명 없이 넘어가면 오해하기 쉬운 부분을 따로 정리했다.

  • 0으로 나누는 percent_change는 None으로 처리한다.
  • 지표가 없거나 None이면 improved와 regressed 대신 n/a가 나온다.
  • 이 파일은 아직 threshold 기반 pass와 fail보다 방향 판정 중심이다.
  • 좋아졌다는 판단은 METRICS에 적힌 direction 정의에 전적으로 달려 있다.
