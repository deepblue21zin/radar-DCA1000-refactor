쉽게 이해하는 모델

처음 보는 사람이 이 파일을 딱 한 문장으로 붙잡아야 할 때의 설명이다.

이 파일은 점 무리 해석기다. detection이 만든 point들을 사람 단위 cluster로 묶고, 밴드 경계 때문에 잘못 갈라진 cluster까지 다시 붙인다.

입력에서 출력까지 단계별 설명

질문에서 정리한 흐름이 큰 줄기는 맞다. 여기에 실제 코드가 하는 입력 정리, fallback range 계산, 경계 병합 조건, confidence 계산까지 더 보면 전체가 완성된다.

1. 입력 포인트를 깨끗하게 정리한다

Step

cluster_points는 각 point의 x, y를 float로 강제 변환하고, v와 score와 range도 유효하지 않으면 0으로 보정한다.

왜 필요한가DBSCAN은 거리 계산 기반이라 숫자형이 깨지면 전체 군집화가 바로 흔들린다.
다음으로 넘기는 것clean point list

2. adaptive band 정의를 정규화한다

Step

normalize_adaptive_eps_bands는 list, tuple, JSON 문자열, compact 문자열 입력을 모두 받아 r_min, r_max, eps, min_samples가 정리된 band dict로 바꾼다.

왜 필요한가사용자가 밴드 설정을 여러 형식으로 줘도 clustering 본체는 항상 같은 구조를 보게 만들기 위해서다.
다음으로 넘기는 것정렬되고 겹치지 않는 adaptive eps bands

3. 각 point가 어느 band에 속하는지 나눈다

Step

_effective_range가 point.range가 없거나 이상할 때는 hypot(x, y)로 range를 계산하고, _range_matches_band가 r_min과 r_max로 band를 판정한다.

왜 필요한가실제 point마다 다른 eps를 써야 가까운 거리와 먼 거리에서 군집 밀도 기준을 다르게 줄 수 있다.
다음으로 넘기는 것band별 point list와 fallback_points

4. DBSCAN용 좌표를 만든다

Step

_build_feature_matrix는 기본적으로 (x, y)를 만들고, velocity feature를 켜면 (x, y, v * weight)로 확장한다.

왜 필요한가같은 위치에 가까워도 속도 차이가 큰 점은 분리하고 싶을 때 velocity를 제3축으로 쓰기 위해서다.
다음으로 넘기는 것feature matrix

5. BFS 방식으로 cluster를 확장한다

Step

_dbscan_labels는 모든 점을 labels=-1로 시작하고, core point를 만나면 seeds 큐를 넓혀 가며 같은 cluster_id를 붙인다.

왜 필요한가DBSCAN의 핵심은 core point 주변의 밀집 연결성을 따라가며 point를 확장하는 것이다.
다음으로 넘기는 것point별 cluster label, noise=-1

6. cluster를 요약 정보로 바꾼다

Step

_summarize_cluster_points는 점수 가중 평균으로 중심 x/y/v를 계산하고, spread_xy와 peak_score를 이용해 confidence를 만든다.

왜 필요한가후단 detection은 개별 point 목록보다 이 cluster가 어디에 있고 얼마나 믿을 만한가라는 요약값이 필요하다.
다음으로 넘기는 것cluster summary dict

7. band 경계에서 갈라진 cluster를 다시 붙인다

Step

_merge_adaptive_boundary_clusters는 인접 band 경계를 공유하고 중심 거리와 경계 근접 조건을 만족하는 cluster 쌍을 다시 병합한다.

왜 필요한가원래 하나의 사람인데 band가 달라 eps가 달라져 인위적으로 쪼개지는 현상을 줄이기 위해서다.
다음으로 넘기는 것최종 public cluster list

놓치기 쉬운 부분

코드를 읽을 때 자주 헷갈리거나, 설명 없이 넘어가면 오해하기 쉬운 부분을 따로 정리했다.

  • visited와 label은 다른 개념이다. visited=True인데 label=-1이면 검사는 했지만 아직 noise 상태라는 뜻이다.
  • labels가 -1로 시작하는 것은 실패가 아니라 DBSCAN의 정상 초기화다. border point는 나중에 다른 core point의 이웃으로 편입될 수 있다.
  • confidence는 확률이 아니라 size, spread, peak_score를 섞은 heuristic 점수다.
  • adaptive band를 쓰더라도 어떤 band에도 안 들어간 point는 fallback_points 경로로 한 번 더 clustering된다.
  • 경계 병합은 shared boundary, centroid distance, boundary proximity를 모두 통과해야 한다.

246# 클러스터 요약값 만들기. 클러스터에 속한 점들의 x, y, v의 가중 평균을 계산하여 클러스터 중심을 구하고, 클러스터의 크기, 확산 정도, 점수 통계 등을 계산하여 클러스터의 특성을 요약하는 딕셔너리를 반환합니다. 또한 adaptive eps 밴드 정보와 경계 병합 여부도 포함할 수 있습니다.
247def _summarize_cluster_points( # 즉, 물체가 하나처럼 보이는 요약 정보를 만든다.
248 c_points: List[dict],
249 label: int,
250 eps: float,
251 min_samples: int,
252 range_band_desc: Optional[str] = None,
253 band_r_min: Optional[float] = None,
254 band_r_max: Optional[float] = None,
255 boundary_merged: bool = False,
256) -> dict:
257 size = len(c_points)
258 if size == 0:
259 raise ValueError("Cannot summarize an empty cluster.")
260
261 weights = [max(float(point.get("score", 0.0)), 0.0) for point in c_points]
262 weight_sum = sum(weights)
263 if weight_sum <= 0.0:
264 weights = [1.0] * size # 점들의 score가 0이하면 weight를 1로 설정
265 weight_sum = float(size)
266 # 가중평균 계산
267 x_mean = sum(float(point["x"]) * weight for point, weight in zip(c_points, weights)) / weight_sum
268 y_mean = sum(float(point["y"]) * weight for point, weight in zip(c_points, weights)) / weight_sum
269 v_mean = sum(float(point.get("v", 0.0)) * weight for point, weight in zip(c_points, weights)) / weight_sum
270 range_vals = [_effective_range(point) for point in c_points]
271 spread_xy = math.sqrt( # 중심으로부터 퍼진 정도를 계산. 작을 수록 점들이 촘촘히 모여 있다는 뜻이다.
272 sum(
273 (math.hypot(float(point["x"]) - x_mean, float(point["y"]) - y_mean) ** 2)
274 for point in c_points
275 ) / float(size)
276 )
277 peak_score = max(float(point.get("score", 0.0)) for point in c_points) # 이 값이 클러스터의 대표 점수, 높을 수록 강한 검출
278 mean_score = sum(float(point.get("score", 0.0)) for point in c_points) / float(size)
279
280 size_score = min(1.0, float(size) / max(float(min_samples), 1.0))
281 spread_score = max(0.0, 1.0 - (spread_xy / max(eps, 1e-6))) # spread가 eps보다 작을 수록 1에 가까워지는 점수. spread가 eps 이상이면 0이 됩니다.
282 score_strength = min(1.0, peak_score / 3.0)
283 confidence = max(0.0, min(1.0, (0.45 * size_score) + (0.3 * score_strength) + (0.25 * spread_score))) # 클러스터의 신뢰도 계산, 너무 퍼져있지 않을때 높은 점수 받음
284
285 cluster = { # 사용자에게 보여줄 정보
286 "x": float(x_mean),
287 "y": float(y_mean),
288 "v": float(v_mean),
289 "size": size,
290 "confidence": float(confidence),
291 "label": label,
292 "spread_xy": float(spread_xy),
293 "mean_score": float(mean_score),
294 "peak_score": float(peak_score),
295 "eps_used": float(eps),
296 "min_samples_used": int(min_samples),
297 "member_points": list(c_points),
298 "_member_points": list(c_points),
299 "_band_r_min": band_r_min,
300 "_band_r_max": band_r_max,
301 "_point_range_min": float(min(range_vals)),
302 "_point_range_max": float(max(range_vals)),
303 }
304 if range_band_desc is not None:
305 cluster["range_band"] = range_band_desc
306 if boundary_merged:
307 cluster["boundary_merged"] = True
308 return cluster

355def _merge_adaptive_boundary_clusters(clusters: List[dict]) -> List[dict]: # adaptive eps 밴드 경계를 공유하는 클러스터들을 병합하여 더 큰 클러스터로 만드는 함수입니다. 각 클러스터의 중심 간 거리가 병합 임계값(eps) 이내이고, 클러스터의 점들이 공유된 밴드 경계 근처에 있는 경우에 병합이 수행됩니다. 병합된 클러스터는 점들의 결합된 요약값으로 다시 계산됩니다. 이 과정을 반복하여 더 이상 병합할 수 있는 클러스터 쌍이 없을 때까지 진행합니다.

382 upper_cluster = right_cluster
383 else:

388 upper_near_boundary = float(upper_cluster["_point_range_min"]) <= shared_boundary + merge_threshold
389 if not (lower_near_boundary and upper_near_boundary):
