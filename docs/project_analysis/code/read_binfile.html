처음 보는 사람이 이 파일을 딱 한 문장으로 붙잡아야 할 때의 설명이다.

이 파일은 오프라인 입력 복원기다. mmWave Studio .bin 파일을 complex radar cube로 되돌려 replay와 offline 분석에 쓰게 만든다.

입력에서 출력까지 단계별 설명

실시간 비교는 장면이 조금만 달라도 흔들리기 쉬우므로, 결국 같은 raw 입력을 다시 돌리는 replay 경로가 필요하다. 그 시작점이 이 파일이다.

1. 필요하면 UDP header를 제거한다

Step

remove_header는 Studio capture에 붙은 헤더 샘플을 삭제해 순수 ADC 데이터만 남긴다.

왜 필요한가헤더가 남아 있으면 이후 reshape가 전부 틀어지기 때문이다.
다음으로 넘기는 것header가 제거된 int16 stream

2. int16를 complex IQ로 묶는다

Step

read_bin_file은 실수와 허수 채널을 묶어 복소수 샘플로 만든다.

왜 필요한가레이더 DSP는 복소수 baseband 데이터를 전제로 한다.
다음으로 넘기는 것complex sample array

3. 장비 모드에 맞게 reshape한다

Step

mode와 tx와 rx 수에 따라 frame, chirp, sample, channel 축으로 재배열한다.

왜 필요한가XWR 계열마다 raw 저장 순서가 달라서 하나의 reshape 규칙으로는 맞지 않기 때문이다.
다음으로 넘기는 것frame x chirp x sample x channel cube

4. DSP와 replay 경로로 넘긴다

Step

복원된 cube는 DSP baseline이나 future replay pipeline의 입력으로 재사용될 수 있다.

왜 필요한가같은 입력으로 전후 알고리즘을 비교해야 의미 있는 A/B가 되기 때문이다.
다음으로 넘기는 것offline radar cube

놓치기 쉬운 부분

코드를 읽을 때 자주 헷갈리거나, 설명 없이 넘어가면 오해하기 쉬운 부분을 따로 정리했다.

  • mode, tx_num, rx_num, sample 수가 하나라도 틀리면 output shape가 완전히 어긋난다.
  • header 제거 여부를 잘못 선택하면 값 자체가 맞아 보여도 축이 틀어질 수 있다.
  • 이 파일은 실시간 파이프라인이 아니라 offline와 replay 입력 계층으로 읽는 편이 맞다.
  • 현재 앱의 기본 경로는 live이지만, 재현 가능한 실험은 결국 이 파일 기반 replay가 필요하다.
