쉽게 이해하는 모델

처음 보는 사람이 이 파일을 딱 한 문장으로 붙잡아야 할 때의 설명이다.

이 파일은 기초 FFT 엔진이다. radar cube를 range-doppler와 range-angle 맵으로 바꿔 후단 검출이 쓸 수 있는 스펙트럼을 만든다.

입력에서 출력까지 단계별 설명

현재 프로젝트의 기본 angle estimator는 FFT 기반이므로, 이 파일에서 나오는 맵 품질이 detection 품질의 바닥을 결정한다.

1. 입력 cube에 window를 건다

Step

Range_Doppler와 Range_Angle 모두 Hanning window를 chirp와 sample 축에 곱한다.

왜 필요한가FFT leakage를 줄여 peak가 더 날카롭게 보이게 하려는 기본 전처리다.
다음으로 넘기는 것windowed radar cube

2. range-doppler 2D FFT를 수행한다

Step

Range_Doppler는 chirp와 sample 축 2D FFT를 수행하고 mode에 따라 raw 또는 abs/shifted 출력을 만든다.

왜 필요한가움직임과 거리 정보는 먼저 range-doppler 평면에서 분리하는 것이 기본이다.
다음으로 넘기는 것RDI cube

3. angle FFT를 추가로 수행한다

Step

Range_Angle은 앞선 2D FFT 결과에 안테나 축 FFT를 적용해 angle 성분을 만든다.

왜 필요한가같은 거리와 속도에 있는 대상들을 각도 차이로 나누기 위해서다.
다음으로 넘기는 것RAI cube

4. 후단이 보기 쉬운 형태로 바꾼다

Step

mode=1이나 mode=2에서는 abs와 fftshift와 flip을 적용해 시각화와 detection에 편한 형태로 바꾼다.

왜 필요한가후단이 복소수 raw 스펙트럼을 직접 다루지 않고, 정렬된 magnitude 맵을 쓰기 때문이다.
다음으로 넘기는 것시각화와 검출용 RDI, RAI magnitude map

놓치기 쉬운 부분

코드를 읽을 때 자주 헷갈리거나, 설명 없이 넘어가면 오해하기 쉬운 부분을 따로 정리했다.

  • 이 파일은 고급 beamforming이 아니라 baseline FFT angle estimator다.
  • mode에 따라 반환 shape와 타입이 달라지므로, 호출부가 어떤 모드를 기대하는지 항상 같이 봐야 한다.
  • windowing이 입력 배열을 in-place로 바꾸기 때문에 copy를 넘기는 호출부 설계가 중요하다.
  • 축 순서와 transpose와 flip을 놓치면 화면은 그럴듯해 보여도 좌표 해석이 틀릴 수 있다.
