쉽게 이해하는 모델

처음 보는 사람이 이 파일을 딱 한 문장으로 붙잡아야 할 때의 설명이다.

이 파일은 좌표계 번역기다. raw IQ를 radar cube, RDI, RAI, ROI 좌표계로 바꿔서 뒤 단계가 같은 언어를 쓰게 만든다.

입력에서 출력까지 단계별 설명

뒤 단계 detection과 tracking은 이 파일이 만든 배열 shape, 축 길이, 거리와 각도 해석을 그대로 믿고 움직인다.

1. cfg를 runtime config로 해석한다

Step

parse_runtime_config는 chirp, sample, tx/rx 수를 읽어 FFT 크기와 range/angle 축을 계산한다.

왜 필요한가입력 신호 길이와 공간 축 정의가 맞아야 이후 모든 bin 해석이 맞는다.
다음으로 넘기는 것RadarRuntimeConfig

2. 프레임 버퍼를 cube로 재배열한다

Step

frame_to_radar_cube는 raw IQ를 chirp, sample, virtual antenna 축으로 재구성한다.

왜 필요한가DSP 함수는 선형 버퍼가 아니라 의미 있는 축 구조를 가진 cube를 필요로 한다.
다음으로 넘기는 것radar cube

3. 정지 clutter를 줄인다

Step

remove_static_clutter는 프레임 평균 성분을 빼서 움직이는 에너지를 더 눈에 띄게 만든다.

왜 필요한가정지 배경이 강하면 움직이는 대상이 묻혀 detection threshold가 흔들린다.
다음으로 넘기는 것clutter가 줄어든 radar cube

4. 시각화와 검출용 맵을 만든다

Step

integrate_rdi_channels와 collapse_motion_rai가 detection에 쓸 range-doppler, range-angle 맵을 만든다.

왜 필요한가후단은 raw 복소수보다 압축된 에너지 맵에서 훨씬 안정적으로 후보를 찾는다.
다음으로 넘기는 것RDI, motion RAI

5. 공간 ROI를 적용한다

Step

apply_cartesian_roi_to_rai는 lateral과 forward 범위를 기준으로 각도 축 일부만 남긴다.

왜 필요한가관심 영역 밖의 포인트를 early filter하면 false positive와 계산 낭비가 줄어든다.
다음으로 넘기는 것ROI가 적용된 angle 정보

놓치기 쉬운 부분

코드를 읽을 때 자주 헷갈리거나, 설명 없이 넘어가면 오해하기 쉬운 부분을 따로 정리했다.

  • shape가 한 축이라도 어긋나면 detection과 tracking이 모두 연쇄적으로 틀어진다.
  • angle axis는 단순 시각화 축이 아니라 DetectionCandidate의 angle_bin과 x/y 계산 기준이다.
  • remove_static_clutter는 항상 좋은 것이 아니라 정지 대상까지 보고 싶다면 정보를 버릴 수도 있다.
  • 향후 Capon 같은 고급 angle estimator를 넣어도 axis와 antenna geometry 해석은 이 계층 이해가 먼저다.

130def frame_to_radar_cube(frame_data, runtime_config):
131 complex_frame = np.reshape(np.asarray(frame_data), [-1, 4])
132 complex_frame = complex_frame[:, 0:2:] + 1j * complex_frame[:, 2::]

134 complex_frame,

163def collapse_motion_rai(rai_cube, guard_bins=1):
164 if rai_cube.ndim != 3:
165 return rai_cube
166
167 motion_cube = np.array(rai_cube, copy=True)
168 center_bin = motion_cube.shape[0] // 2
169 lower = max(center_bin - guard_bins, 0)
170 upper = min(center_bin + guard_bins + 1, motion_cube.shape[0])
171 motion_cube[lower:upper, :, :] = 0
172 return np.max(motion_cube, axis=0)

180def apply_cartesian_roi_to_rai(
181 rai_map,
182 runtime_config,
183 lateral_limit_m,
184 forward_limit_m,
185 min_forward_m=0.0,
186):
187 range_axis = runtime_config.range_axis_m[:, np.newaxis]
188 angle_axis = runtime_config.angle_axis_rad[np.newaxis, :]
189 x_axis = range_axis * np.sin(angle_axis)
190 y_axis = range_axis * np.cos(angle_axis)
191 roi_mask = (
192 (np.abs(x_axis) <= lateral_limit_m)
193 & (y_axis >= min_forward_m)
194 & (y_axis <= forward_limit_m)
195 )
196 masked_rai = np.where(roi_mask, rai_map, 0)
197 min_radial_bin = radial_bin_limit(runtime_config, min_forward_m) if min_forward_m > 0 else 0
198 max_radial_distance = np.sqrt((lateral_limit_m ** 2) + (forward_limit_m ** 2))
199 max_radial_bin = radial_bin_limit(runtime_config, max_radial_distance)
200 return masked_rai[min_radial_bin:max_radial_bin, :]
