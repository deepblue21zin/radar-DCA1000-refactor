왜 필요한가옛 세션이나 일부 로그가 없는 세션도 최대한 읽혀야 실험 기록을 잃지 않는다.
다음으로 넘기는 것processed_records, render_records, session_meta

2. 숫자 집계 함수를 준비한다

왜 필요한가알고리즘 처리 성능과 사용자 체감 렌더 성능을 같은 값으로 섞으면 원인 분석이 어려워진다.

놓치기 쉬운 부분

코드를 읽을 때 자주 헷갈리거나, 설명 없이 넘어가면 오해하기 쉬운 부분을 따로 정리했다.

  • render 로그가 없으면 legacy status_log.jsonl을 fallback으로 읽는다.
  • 이 파일이 계산하는 success rate는 label 없는 운영 지표이지 정답 기반 accuracy 지표는 아니다.
  • processed와 render를 분리해 보는 것이 이 파일의 가장 중요한 가치다.
  • summary schema가 바뀌면 session_compare와 장기 추세 분석도 함께 손봐야 한다.

44def _quantile(values, q: float):
45 if not values:
46 return None
47 ordered = sorted(float(value) for value in values)
48 if len(ordered) == 1:
49 return ordered[0]
50 position = (len(ordered) - 1) * q
51 lower = int(math.floor(position))
52 upper = int(math.ceil(position))
53 if lower == upper:
54 return ordered[lower]
55 lower_value = ordered[lower]
56 upper_value = ordered[upper]
57 weight = position - lower
58 return lower_value + ((upper_value - lower_value) * weight)
