상태 유지

확정 정책

다중 타깃 정합

가까운 후보가 여러 개일 때 어떤 detection을 어떤 track에 잇는지 계산한다.

쉽게 이해하는 모델

처음 보는 사람이 이 파일을 딱 한 문장으로 붙잡아야 할 때의 설명이다.

이 파일은 사람 ID 유지기다. 프레임마다 detection을 받아 기존 track과 잇고, tentative를 confirmed로 승격시키거나 오래된 track을 정리한다.

입력에서 출력까지 단계별 설명

detection은 한 프레임짜리 점 정보이고, tracking은 시간축 가설 관리다. 이 파일은 같은 사람이 다음 프레임에서도 같은 ID로 이어지게 만드는 정책을 모아 둔 곳이다.

1. tracker 정책을 초기화한다

Step

MultiTargetTracker.__init__은 process_var, measurement_var, gate, doppler 조건 등 추적 정책을 저장한다.

왜 필요한가추적 품질은 알고리즘보다 먼저 정책 숫자에 크게 좌우된다.
다음으로 넘기는 것프레임마다 재사용되는 tracker 상태

2. 모든 기존 track을 예측한다

Step

Kalman filter로 현재 프레임 시점의 예상 위치와 속도를 계산한다.

왜 필요한가매칭은 현재 detection이 어디에 있을지 예상한 위치를 기준으로 해야 안정적이다.
다음으로 넘기는 것예측된 track 상태

3. detection과 track 사이 비용을 만든다

Step

_associate는 거리와 doppler 차이를 비용으로 만들고, gate 밖 후보는 배제한다.

왜 필요한가가까워 보이는 detection이라도 속도나 상태가 너무 다르면 같은 사람으로 이어 붙이면 안 된다.
다음으로 넘기는 것cost matrix와 유효 후보 쌍

4. 최적 매칭을 선택한다

Step

Hungarian assignment로 여러 detection과 여러 track의 최적 짝을 고른다.

왜 필요한가다중 타깃 상황에서 탐욕적으로 매칭하면 ID switch가 쉽게 난다.

왜 필요한가새 대상이 들어오거나 잠깐 가려지는 상황을 동시에 처리해야 하기 때문이다.

왜 필요한가내부 Kalman 상태와 사용자에게 보여 줄 값은 다르므로 공개용 구조가 따로 필요하다.

놓치기 쉬운 부분

코드를 읽을 때 자주 헷갈리거나, 설명 없이 넘어가면 오해하기 쉬운 부분을 따로 정리했다.

  • tentative track은 아직 확정된 사람이 아니다. confirm_hits 조건을 통과해야 confirmed가 된다.
  • association gate 하나만 있는 것이 아니라 lost_gate_factor와 tentative_gate_factor로 상태별 gate가 다시 조절된다.
  • doppler gating은 multi-target 상황에서 ID switch를 줄이는 중요한 축이다.
  • confidence는 표시용 점수이기도 하지만 measurement noise scaling에도 간접적으로 영향을 준다.
  • tracker가 좋아도 detection이 흔들리면 birth와 death가 잦아지므로 detection과 함께 봐야 한다.

18class _SimpleKalmanFilter:

21 def __init__(self, dim_x: int, dim_z: int):
22 self.dim_x = dim_x
23 self.dim_z = dim_z
24 self.x = np.zeros((dim_x, 1), dtype=float)
25 self.F = np.eye(dim_x, dtype=float)
26 self.H = np.zeros((dim_z, dim_x), dtype=float)
27 self.P = np.eye(dim_x, dtype=float)
28 self.Q = np.eye(dim_x, dtype=float)
29 self.R = np.eye(dim_z, dtype=float)
30
31 def predict(self) -> np.ndarray:
32 self.x = self.F @ self.x
33 self.P = self.F @ self.P @ self.F.T + self.Q
34 return self.x
35
36 def update(self, z: np.ndarray) -> np.ndarray:
37 y = z - (self.H @ self.x)
38 pht = self.P @ self.H.T
39 s = self.H @ pht + self.R
40 try:
41 k = np.linalg.solve(s, pht.T).T
42 except np.linalg.LinAlgError:
43 k = pht @ np.linalg.pinv(s)
44 self.x = self.x + (k @ y)
45 identity = np.eye(self.dim_x, dtype=float)
46 kh = k @ self.H
47 self.P = (identity - kh) @ self.P @ (identity - kh).T + k @ self.R @ k.T
48 return self.x
