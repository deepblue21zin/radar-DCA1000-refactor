쉽게 이해하는 모델

처음 보는 사람이 이 파일을 딱 한 문장으로 붙잡아야 할 때의 설명이다.

이 파일은 후보 심사관이다. RDI와 RAI에서 에너지 피크를 찾고, 각도와 공간 조건을 통과한 점만 DetectionCandidate로 만든다.

입력에서 출력까지 단계별 설명

한 프레임의 모든 에너지 피크를 다 객체로 취급하지 않고, 여러 단계 조건을 거쳐 tracker가 믿을 수 있는 후보만 남긴다.

1. RDI 관심 구간만 잘라 쓴다

Step

detect_targets는 min과 max range bin 안쪽만 사용하고 zero doppler 주변은 guard bin으로 비운다.

왜 필요한가정지 clutter와 관심 밖 거리 구간을 줄여야 moving target 후보가 더 또렷해진다.
다음으로 넘기는 것정제된 rdi_work와 power_map

2. CFAR와 global floor를 합쳐 threshold를 만든다

Step

cfar_threshold_2d와 global_quantile이 함께 threshold_map을 만들고, local maxima만 후보로 남긴다.

왜 필요한가배경 대비 강한 점과 전체 분포 대비 강한 점을 동시에 요구해야 잡음에 덜 흔들린다.
다음으로 넘기는 것candidate_indices

3. 각도 프로필을 한 번 더 심사한다

Step

각 candidate마다 RAI angle profile을 보고 ROI 안인지, local peak인지, angle contrast가 충분한지 확인한다.

왜 필요한가RDI에서만 강한 점은 있어도 실제 각도 분해가 불안정하면 tracker 입력으로 쓰기 어렵다.
다음으로 넘기는 것candidate_pool

4. Cartesian 기준으로 묶는다

Step

_cluster_detection_candidates는 x와 y 좌표가 가까운 점을 dbscan_cluster로 묶는다.

왜 필요한가한 사람에서 range-doppler peak가 여러 개 나와도 detection 수를 사람이 이해할 수 있는 수로 줄여야 한다.
다음으로 넘기는 것cluster summary

5. 클러스터를 DetectionCandidate로 바꾼다

Step

cluster 중심 x/y와 seed 점 score를 이용해 range_bin, angle_bin, doppler_bin을 다시 계산한다.

왜 필요한가tracker는 맵 픽셀 집합이 아니라, 좌표와 점수를 가진 후보 리스트를 필요로 한다.
다음으로 넘기는 것DetectionCandidate list

6. 점수 순으로 정렬해 넘긴다

Step

최종 후보는 score, rdi_peak, rai_peak 기준으로 정렬되어 tracker로 전달된다.

왜 필요한가후단에서 상위 후보를 먼저 다루거나 로그를 볼 때 일관된 우선순위가 필요하다.

놓치기 쉬운 부분

코드를 읽을 때 자주 헷갈리거나, 설명 없이 넘어가면 오해하기 쉬운 부분을 따로 정리했다.

  • DetectionCandidate 수는 raw peak 수와 다르다. clustering과 angle filter 때문에 줄어든 결과다.
  • cfar_scale, global_quantile, angle_quantile은 서로 독립 파라미터가 아니라 함께 detection 민감도를 만든다.
  • min_cartesian_separation_m은 range-doppler가 아니라 x와 y 공간에서 두 후보를 얼마나 떨어뜨릴지 정하는 값이다.
  • allow_strongest_fallback은 후보가 없을 때 가장 강한 점 하나를 강제로 살리는 안전장치다.
  • 지금 detection score는 확률값이 아니라 heuristic 조합값이므로 절대치보다 상대 비교로 보는 편이 안전하다.

37 padded = np.pad(power_map, 1, mode='constant', constant_values=-np.inf)
38 center = padded[1:-1, 1:-1]
39 maxima_mask = np.ones((rows, cols), dtype=bool)
